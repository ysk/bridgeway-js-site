jQueryの書き方で
モダン開発の体験を

覚えるのは $ を $$ にするだけ。
慣れたjQueryの書き方で、state / computedが加わります。
モダン開発の体験をjQueryの書き心地で。

チュートリアルを始める サンプルを見る
$npm install bridgey
CDNhttps://cdn.jsdelivr.net/npm/bridgey/dist/bridgey.vue.js

覚えた書き味（$$ / state / mount）は変わらない。下のエンジン（Svelte / Vue）が進化しても、あなたのコードはそのまま。

なぜ bridgey か

レガシーの手を止めずに、"変えたら反映"を。

jQueryが分かるなら、もう入口に立っています。

jQueryに、リアクティビティ

state／computed で「変えたら自動で反映」。合計・絞り込み・出し分けを、もう手で書き直さない。

ビルドなしで始まる

<script> 一本で $$／state／computed。npmもツールチェーンも後回しでいい。

学習コスト最小

触るのは $$（jQuery風）・state・computed・mount。「jQuery＋α」で始められる。

複雑になったら本物へ

重い画面は mount で本物のSvelte／Vueへ引き上げ。DOM差分も後片付けもエンジン任せ。フォーカスも入力途中も飛ばない。

CSPセーフ（Svelte）

Svelteは式を eval せずビルド時コンパイル。script-src 'self' の企業／WordPress環境でも動く。
※VueのCDN／既定ビルドは template を実行時コンパイル（new Function）するため unsafe-eval が必要。

CLIで一発

brg init でフレームワークを選ぶだけ。配線済みスタータが生成される。

Concept

モダンな技術を、限りなく楽に。

学習コストの削減。そして「翻弄されない」こと。

背景

作者はかつてフロントエンドエンジニアとして、jQueryからVueへの移行が難しくとん挫しました。モダン技術に触れられず焦りつつ、日々の業務はレガシー——そんな人は今も多いはず。あの頃の自分を救うための橋を架けたい。それが bridgey の出発点です。

誰のためか

・レガシー環境で日々jQueryを保守している人
・WordPress等でモダンFWを導入する機会がない現場
・「jQueryなら分かる」人

全書き換えを迫らず、書き味を残したまま、本物のモダンへ段階的に手を慣らしていけます。

究極的には——もう新しいフロントエンドフレームワークに翻弄されない。

使い続けてもらうための道具ではありません。

bridgey は通過点です。モダンフレームワークへ “渡り切る” ための橋。ゴールは、あなたが bridgey を卒業すること。
橋を渡り終えたら、捨ててくれて構いません。気づいたら本物のモダンフレームワークを書けている。それがこのライブラリの目的です。

$$ で触る

jQueryの手癖のまま、イベント・クラス・テキストを動かす。

state / computed で反応させる

「変えたら自動で反映」。手で書き直す作業から解放される。

mount で本物の部品を載せる

重い画面だけ本物のSvelte／Vueへ。DOM差分も後片付けもエンジン任せ。

やがて .svelte / .vue を自分で書く

いつのまにか本物を書いている。このライブラリの卒業。

Install

3分ではじめる

CLIが、選んだフレームワークに配線済みのスタータを生成します。

フレームワークを選んでスタータ生成 
npx brg init my-app
? フレームワークを選択:  1) svelte   2) vue
# → App.(svelte|js) / main.js / build.mjs / index.html を生成
依存をまとめて導入 
cd my-app
npm install          # bridgey と svelte(or vue) が一緒に入る
ビルドして開く 
npm run build        # main.js(+部品) → bundle.js
npx serve .          # index.html を開く

ダウンロード

2つの入り方

きちんと作るなら npm。今すぐ試すなら <script> 一本（ビルド不要）。

npm（本格運用）
npx brg init my-app   # svelte / vue を選ぶ
cd my-app
npm install           # bridgey と svelte(or vue) を一括で
npm run build         # .svelte/.vue → bundle.js

brg init が package.json に svelte(or vue) まで書いた配線済みスタータを生成。ライブラリを手で入れ足す必要はありません。

<script>（ビルド不要・CDN）
<!-- Vueフルビルド同梱。mountまで全部動く -->
<script src="…/bridgey.vue.js"></script>
<script>
  const App = { data: () => ({ n: 0 }),
    template: `<button @click="n++">{{ n }}</button>` };
  mount(App, { target: "#app" });
</script>

Vue公式のCDNフルビルドと同じ発想。$$ / state / mount が window に。軽量な bridgey(Svelte・$$/state中心)版もあります。
