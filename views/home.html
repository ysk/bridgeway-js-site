jQuery感覚のまま、モダンへ。

jQuery感覚で、
モダン開発。

覚えるのは $ を $$ にすることだけ。いつものjQueryの書き味そのままで、裏では本物のSvelte／Vueが動く。学習コストを限りなく下げて、モダンフロントエンドへ。

はじめる サンプルを見る
$npm install bridgeway

もう新しいフロントエンドフレームワークに翻弄されない — エンジンは差し替え式。

なぜ bridgeway か

レガシーの手を止めずに、本物の恩恵を。

jQueryが分かるなら、もう入口に立っています。

本物のフレームワーク

構造・制御構文・DOM差分は本物のSvelte／Vueに委譲。外科手術的な更新で、フォーカスも入力途中も飛ばない。

エンジン差し替え式

利用者の $$/state/mount は不変のまま、下をSvelte↔Vueに交換。フレームワークの寿命に振り回されない。

学習コスト最小

触るのは3つだけ：$$（jQuery風）・state・mount。「jQuery＋α」で始められる。

CSPセーフ

式は eval せずビルド時コンパイル。script-src 'self' の企業／WordPress環境でも動く。

自動で後片付け

mount().destroy() で購読もDOMもフレームワークが破棄。長時間開く画面でもリークしない。

CLIで一発

bridgeway init でフレームワークを選ぶだけ。配線済みスタータが生成される。

Concept

モダンな技術を、限りなく楽に。

学習コストの削減。そして「翻弄されない」こと。

背景

作者はかつてフロントエンドエンジニアとして、jQueryからVueへの移行が難しくとん挫しました。モダン技術に触れられず焦りつつ、日々の業務はレガシー——そんな人は今も多いはず。あの頃の自分を救うための橋を架けたい。それが bridgeway の出発点です。

誰のためか

・レガシー環境で日々jQueryを保守している人
・WordPress等でモダンFWを導入する機会がない現場
・「jQueryなら分かる」人

全書き換えを迫らず、書き味を残したまま、本物のモダンへ段階的に手を慣らしていけます。

究極的には——もう新しいフロントエンドフレームワークに翻弄されない。

Install

3分ではじめる

CLIが、選んだフレームワークに配線済みのスタータを生成します。

インストール 
# bridge本体を入れる
npm install bridgeway
フレームワークを選んでスタータ生成 
npx bridgeway init my-app
? フレームワークを選択:  1) svelte   2) vue
# → App.(svelte|js) / main.js / build.mjs / index.html を生成
ビルドして開く 
cd my-app
npm install
npm run build        # main.js(+部品) → bundle.js
npx serve .          # index.html を開く

2つの入り方

きちんと作るなら npm。今すぐ試すなら <script> 一本（ビルド不要）。

npm（本格運用）
npm install bridgeway
# 使う方だけ(optional peer)
npm install svelte   # or  npm install vue

bridgeway init で配線済みスタータ。部品は .svelte / .vue をビルドします。

<script>（ビルド不要・CDN）
<!-- Vueフルビルド同梱。mountまで全部動く -->
<script src="…/bridge.vue.js"></script>
<script>
  const App = { data: () => ({ n: 0 }),
    template: `<button @click="n++">{{ n }}</button>` };
  mount(App, { target: "#app" });
</script>

Vue公式のCDNフルビルドと同じ発想。$$ / state / mount が window に。軽量な bridgeway(Svelte・$$/state中心)版もあります。
