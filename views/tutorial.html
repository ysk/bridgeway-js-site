チュートリアル

jQueryが分かれば、もう半分終わっています。$ を $$ にするところから始めて、命令的（手で更新）→ 宣言的（状態を変えるだけ）へ、同じ小さなアプリを育てながら少しずつ手を慣らします。写経して動かせる9章立てです。

Ch 1–2jQueryの手癖そのまま Ch 3–4state / computed Ch 5–7一覧・委譲・束縛 Ch 8–9送信・本物の部品

覚える道具はたった5つ：$$ / state / computed / mount / useEngine。この章を終える頃には、全部を一度は自分の手で書いています。

このチュートリアルで作るもの

お題は誰もが一度は書く 「今日のタスク」リスト。追加・完了・削除・残り件数・サーバ保存まで、jQueryの書き味のまま作り、最後に本物のSvelte部品へ“卒業”させます。

できあがる機能

・入力して Enter で追加
・クリックで完了トグル
・✕ で削除（フェード付き）
・「残り 3 件」を自動表示
・ボタンでサーバへ保存

身につくこと

・$$ のDOM操作・イベント委譲
・state による自動再描画
・computed の派生状態
・bindText/bindClass 束縛
・$$.post と mount

各章のコードは前の章の続きです。上から順に写経していけば、最後に1本のアプリになります。完成形に近い動くデモは Examples の ToDo にあります。

準備 — 2つの入り口

環境は好きな方でOK。まず触ってみたいだけなら ①（ビルド不要）。きちんと作るなら ②（npm）。このチュートリアル本文は ① のグローバル前提で進めます（import は不要）。

① <script> 一本（ビルド不要・今すぐ）

<!-- $$ / state / computed / mount が window に載る -->
<script src="https://cdn.jsdelivr.net/npm/bridgey/dist/bridgey.vue.js"></script>
<script>
  useEngine(vueEngine);   // state/computed/mount を使うので最初に一度
  // ここから下、この章のコードを書いていく
</script>

② npm（本格運用） — 各章の頭に次の1行があると考えてください。

import { $$, state, computed, mount, useEngine } from "bridgey";
import { svelteEngine } from "bridgey/engines/svelte.js";
useEngine(svelteEngine);

HTMLはこれだけ用意しておきます。

<input id="new" placeholder="やることを入力…" />
<ul id="list"></ul>
<p>残り <span id="remain">0</span> 件</p>
<button id="save">サーバへ保存</button>
やってみよう：①のHTMLをコピーして開き、コンソールに $$("#list") と打ってみてください。要素が取れれば準備OKです。
1

$ を $$ に — 手癖はそのまま

ゴール：入力欄で Enter を押したらコンソールに文字を出す。jQueryと同じ書き方で。

bridgeyの入口 $$ は、jQueryの $ とほぼ同じ。セレクタで取得し、.on() でイベントを張り、チェーンでつなげます。置き換えるのは記号1つだけです。

これまで（jQuery）
$("#new").on("keydown", function (e) {
  if (e.key === "Enter") {
    console.log(this.value);
  }
});
これから（bridgey）
$$("#new").on("keydown", function (e) {
  if (e.key === "Enter") {
    console.log(this.value);
  }
});

this がイベント発火元の生DOMを指すのも、function(){} で書くのも、まったく同じ。短縮メソッド（.click() 等）や名前空間（.on("scroll.myui")）も使えます。

やってみよう：手元のjQueryスニペットを1つ、$→$$ にして貼り付けてみましょう。DOM操作・イベント・走査ならほぼそのまま動きます。
2

1件追加する — まずは命令的に

ゴール：Enter で <li> をリストに追加する。「手で append」する、いつものやり方から。

状態管理はまだ登場させません。jQuery脳のまま、要素を作って追加します。

$$("#new").on("keydown", function (e) {
  if (e.key !== "Enter" || !this.value.trim()) return;
  // jQuery定番：文字列から要素を作って append、入力欄は空に
  $$("#list").append(`<li><span>${this.value}</span> <button class="del">✕</button></li>`);
  this.value = "";
});

動きます。でも——このやり方だと「今いくつ残っているか」を知りたくなるたびに、DOMを数えて、表示も手で書き換えることになります。件数表示・完了状態・並び替え…と要求が増えるほど、「本当の状態」がDOMのあちこちに散らばっていきます。ここがjQuery開発のしんどさでした。

次の章から、この「散らばり」を state に1か所へ集めます。DOMは状態の写し、という考え方へ切り替えていきましょう。
3

state — 値を変えれば表示が追う

ゴール：タスクを配列の state にまとめ、件数表示を「手で更新しない」ようにする。

state(初期値) はリアクティブな箱です。.value で読み書きし、.subscribe() で「変わったら実行」を登録します（登録時にも即実行されるので初期表示も任せられます）。

// タスクの「本当の状態」はここ1か所に集める
const todos = state([]);  // [{ id, text, done }]
let seq = 0;

$$("#new").on("keydown", function (e) {
  if (e.key !== "Enter" || !this.value.trim()) return;
  // ★配列は“新しい参照を代入”して更新する（ここ重要）
  todos.value = [...todos.value, { id: ++seq, text: this.value.trim(), done: false }];
  this.value = "";
});

// 件数表示は“手で書かない”。todos が変わるたび自動で走る
todos.subscribe((list) => $$("#remain").text(list.length));
つまずきポイント：配列・オブジェクトは push などの破壊的変更では通知されません。todos.value = [...todos.value, x] のように新しい参照を代入してください。「値を差し替える」のがモダンの作法です。

もう #remain を手で書き換えるコードはどこにもありません。状態を変えるだけで、表示がついてきます。これが宣言的ということです。

4

computed — 派生する値は計算させる

ゴール：「残り＝未完了の件数」を、todos から自動で導く。

「残り件数」は todos から計算で出せる値です。こういう派生値は自分で管理せず、computed に任せます。依存（ここでは todos）が変われば自動で計算し直されます。

// 関数の中で使った state を自動で追跡してくれる
const remain = computed(() => todos.value.filter((t) => !t.done).length);

remain.subscribe((n) => $$("#remain").text(n));

これで「残り」は常に正しい値になります。done を切り替えても、削除しても、追加しても——どこにも「残りを再計算する」コードを書かずに。

computed は読み取り専用。依存を明示する書き方 computed(todos, (list) => …) もありますが、まずは関数だけ渡す自動追跡を覚えれば十分です。
5

一覧を state から描く

ゴール：「追加時に append」をやめ、todos が変わったら一覧まるごとを描き直す方式にそろえる。

状態が1か所にあるので、描画も1か所にまとめられます。「状態 → HTML」を1つの関数にして、subscribe で呼ぶだけ。追加・完了・削除、どの操作でも同じ描画が走ります。

todos.subscribe((list) => {
  $$("#list").html(
    list.map((t) =>
      `<li data-id="${t.id}" class="${t.done ? 'done' : ''}">` +
      `<span class="txt">${t.text}</span> <button class="del">✕</button></li>`
    ).join("")
  );
});
やってみよう：Chapter 2 の「append する行」はもう不要です。消しても、Enter でちゃんと増えます（状態が変わって再描画されるから）。DOMは状態の写しを体感しましょう。
ユーザー入力を html() に流すときはエスケープを。実務では text() や置換でサニタイズしてください（本物の部品＝Svelte/Vueなら自動でエスケープされます。これも“卒業”の動機になります）。
6

完了・削除 — イベント委譲が効く

ゴール：行のクリックで完了トグル、✕ で削除。再描画されても動くように。

一覧は毎回描き直されるので、個々の <li> に直接 .on() すると描き直した瞬間に外れます。ここで イベント委譲の出番。親（#list）に1回張れば、中の要素が入れ替わっても効き続けます。jQueryでお馴染みの手ですね。

const byId = (el) => Number($$(el).closest("li").data("id"));

$$("#list")
  // 行文字をクリック → done を反転（this=クリックされた .txt）
  .on("click", ".txt", function () {
    const id = byId(this);
    todos.value = todos.value.map((t) => t.id === id ? { ...t, done: !t.done } : t);
  })
  // ✕ → フェードしてから状態から除く（jQueryらしく見た目も気持ちよく）
  .on("click", ".del", function () {
    const id = byId(this);
    $$(this).closest("li").fadeOut(150);
    setTimeout(() => (todos.value = todos.value.filter((t) => t.id !== id)), 160);
  });

.closest()・.data()・this・.fadeOut()——全部jQueryの手癖のまま。状態の更新方法だけがモダンです。CSSで li.done .txt{ text-decoration:line-through } とすれば、完了表示も再描画に自動で追従します。

7

束縛 — subscribe すら省く

ゴール：「残り件数」の subscribe を bindText に置き換え、後片付けも意識する。

state を要素へ直接つなぐ束縛メソッドがあります。subscribe して .text() する定番を、1行に短縮できます。

Chapter 4 の書き方
remain.subscribe((n) =>
  $$("#remain").text(n)
);
束縛で1行に
$$("#remain").bindText(remain);
// 加工も渡せる：
// .bindText(remain, (n) => n + " 件")

他にも bindClass（真偽でクラス付け外し）・bindAttr・bindValue（入力と双方向）があります。張った購読は内部にたまり、画面を閉じる時に $$(…).dispose() でまとめて解除できます（長時間開く画面のリーク対策）。

// 例：残り0なら「完了！」バッジを光らせる
$$("#remain").bindClass("all-done", remain, (n) => n === 0);
8

サーバへ保存 — $$.post

ゴール：「保存」ボタンで現在のタスクをAPIへ送る。$.ajax の手触りのまま。

通信は $$.ajax / $$.get / $$.post。中身は fetch ですが、jQuery流の .done()/.fail()/.always() でも await でも書けます。オブジェクトを渡すと自動でJSON送信、JSON応答は自動パースされます。

// async/await 版
$$("#save").on("click", async function () {
  $$(this).prop("disabled", true).text("保存中…");   // jQueryそのまま
  try {
    const res = await $$.post("/api/todos", { items: todos.value });
    $$(this).text("保存しました ✓");
  } catch (err) {
    $$(this).prop("disabled", false).text("再試行");
  }
});

// jQuery流チェーンが好みならこちらでも同じことができる
$$.post("/api/todos", { items: todos.value })
  .done((res) => console.log(res))
  .fail((err) => console.error(err))
  .always(() => console.log("完了"));
やってみよう：本物のエコーAPIで試すなら https://jsonplaceholder.typicode.com/posts を宛先に。実際に動く送信例は Examples のフォームにあります。
9

卒業 — 本物の部品へ載せ替える

ゴール：「手で html() して描き直す」一覧を、本物のSvelte部品に置き換える。差分更新・自動エスケープ・キー付き並び替えを“タダで”手に入れる。

ここまでで $$ と state/computed は手に馴染んだはず。複雑さが増してきたら、描画は本物のフレームワークに委ねます。利用者側は mount するだけ。keyed な差分更新・フォーカス保持・XSS対策は、実績あるエンジンがやってくれます。

<!-- Todo.svelte（構造・制御構文・差分はSvelteの領分） -->
<script>
  export let items = [];
  export let onToggle, onRemove;   // 外へはコールバックで渡す
</script>
<ul>
  {#each items as t (t.id)}          // ← キー付き＝賢い差分更新
    <li class:done={t.done}>
      <span on:click={() => onToggle(t.id)}>{t.text}</span>
      <button on:click={() => onRemove(t.id)}>✕</button>
    </li>
  {/each}
</ul>
// main.js — mount して、props を更新するだけ
const app = mount(Todo, { target: "#list", props: { items: todos.value, onToggle, onRemove } });
todos.subscribe((items) => app.set({ items }));   // state が変われば部品へ反映

// 画面を閉じる時：購読もDOMも片付く
// app.destroy();

入力欄・保存ボタン・残り件数といった“まわり”は $$ のままでOK。複雑なところだけ本物の部品へ——これが段階移行の勘所です。動く完成デモは Examples の ToDo を開いてください。

Svelteは .svelte のコンパイルにビルドが要ります（rollup-plugin-svelte）。brg init なら配線済みのスタータが一発で生成されます。ビルドを避けたいなら、Vueフルビルド同梱の <script> 版で template 文字列の部品を mount できます。
