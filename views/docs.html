ドキュメント

利用者が覚えるのは $$ / state / computed / mount / useEngine の5つ。$$ はjQuery定番メソッドを一通り備え、エンジン(Svelte/Vue)に依らず共通です。

bridgey とは

jQueryの書き味を残したまま、本物のモダンフレームワーク(Svelte、将来はVue等)の恩恵を受けるための薄いレイヤーです。画面の構造・制御構文・DOM差分・リアクティビティ・後片付けは実績あるエンジンに委譲し、bridgey自身は次の2つだけを担います。

  • $$（命令的グルー） … jQueryの手癖そのままのDOM操作・イベント。既存の手が動く入口。
  • state / computed / mount … リアクティブ状態と、部品を載せる最小ライフサイクル（下のエンジンに委譲）。

インストール

本格運用は npm。今すぐ試すなら <script>（配布参照）。

npm install bridgey
# 使うフレームワークだけ(optional peer)
npm install svelte     # もしくは  npm install vue

はじめの一歩

$ を $$ にするだけ。まずは命令的に、慣れたら宣言的に。

import { $$, state, computed } from "bridgey";
import { svelteEngine } from "bridgey/engines/svelte.js";
useEngine(svelteEngine); // state/computed を使うなら最初に一度

// 命令的(jQueryそのまま)
$$("#btn").on("click", () => $$("#msg").text("clicked").addClass("on"));

// 宣言的(状態を変えるだけ)
const count = state(0);
$$("#plus").on("click", () => count.value++);
$$("#n").bindText(count);

3つの考え方

考え方意味
薄い皮bridgeyはフレームワークではなくヘルパー。ルーター/スコープCSS等は持たず、最小の mount/destroy だけ。
エンジン差し替えstate/computed/mount は現在のエンジンに委譲。useEngine でSvelte↔Vueを選ぶ。利用者コードは不変。
命令的→宣言的まず $$ の手で書き、慣れたら state 束縛へ。同じ画面で少しずつ移行できる。

$$( ) が受け取れるもの

jQueryの $ 相当。名前衝突を避けて $$ を既定にしています。

呼び方意味
$$("#id .cls")CSSセレクタで取得
$$("<li class='x'>…</li>")HTML文字列から要素を生成
$$(node) / $$(nodeList) / $$(array)DOMノード・NodeList・配列・jQueryオブジェクトを包む
$$(otherBridgey)別の $$ を複製
$$(fn)DOM ready（$(function(){…}) の手癖）
$$(() => {                       // DOMContentLoaded 後に実行
  const $li = $$("<li class='item'>新規</li>"); // 生成
  $li.appendTo("#list");
});

内容: text / html / val / attr …

引数なしで取得、引数ありで設定（設定は this を返しチェーン可）。

メソッド説明
.text(v?)textContent の取得/設定
.html(v?)innerHTML の取得/設定
.val(v?)フォーム値の取得/設定
.attr(name, v?) / .attr({…})属性の取得/設定（オブジェクトで一括）
.removeAttr(name)属性の削除
.prop(name, v?)プロパティ（checked/disabled 等）の取得/設定
.data(key, v?)data-* の取得/設定
$$("#link").attr({ href: "/next", title: "次へ" });
const checked = $$("#agree").prop("checked");
$$("#row").data("id", 42);

.toggleClass("a b", force?)各クラスを付け外し（forceで強制）
.hasClass("a")いずれかが持つか（真偽）

$$("#panel").slideToggle();

メソッド説明
.width() / .height()表示サイズ(px)

.scrollTop(v?)スクロール位置の取得/設定

イベント

直接バインドとイベント委譲、複数イベント、名前空間、短縮メソッドに対応します。

メソッド説明
.on("click", fn)直接バインド
.on("click", ".sel", fn)イベント委譲（this=一致要素。再描画に強い）
.on("click focus", fn)複数イベント（空白区切り）
.on("click.myui", fn)名前空間つき
.off() / .off("click") / .off(".myui") / .off("click", fn)解除（全 / 種別 / 名前空間 / 特定関数）
.one("click", fn)一度だけ
.trigger("click", detail?)発火（detailは e.detail へ）
.click(fn?) / .change() / .submit() …短縮（引数ありでバインド、無しで発火）
.hover(enter, leave?)mouseenter / mouseleave
.ready(fn)DOM ready

短縮メソッド: click dblclick change submit keydown keyup keypress mousedown mouseup mouseenter mouseleave mouseover mouseout input scroll contextmenu

// 名前空間: プラグインの後片付けが一発
$$(document).on("scroll.myui", onScroll).on("resize.myui", onResize);
$$(document).off(".myui"); // myui 名前空間をまとめて解除

リアクティブ束縛（$$ × state）

state をDOMに束縛します。張った購読は内部に溜まり、.dispose() で一括解除できます（リーク対策）。

メソッド説明
.bindText(state, format?)テキストを束縛。変化で自動更新
.bindClass(name, state, pred?)真偽でクラス付け外し
.bindAttr(name, state, format?)属性を束縛
.bindValue(state)input と双方向束縛
.dispose()この $$ が張った購読を全解除
const n = state(0);
$$("#n").bindText(n).bindClass("big", n, (v) => v >= 10);
$$("#name").bindValue(state("")); // 双方向

state(initial)

リアクティブな状態。.value で読み書き、.update(fn) で関数更新、.subscribe(cb) で購読（解除関数を返す。初回も即時発火）。

const count = state(0);
count.value++;                 // 束縛先が自動更新
count.update((n) => n + 1);
const off = count.subscribe((v) => console.log(v)); // off() で解除
配列/オブジェクトは新しい参照を代入して更新します（list.value = [...list.value, x]）。ミューテートだけでは通知されません。

computed( )

派生状態（読み取り専用）。依存が変わると自動再計算。2つの書き方があります。

// (A) 関数だけ → 依存を自動追跡（おすすめ）
const visible = computed(() =>
  filter.value === "done" ? todos.value.filter((t) => t.done) : todos.value
);

// (B) 依存を明示
const double = computed(count, (n) => n * 2);
const total  = computed([a, b], (x, y) => x + y);

mount(Component, options)

本物のコンポーネントを載せ、ハンドルを返します。options は { target, props }（target はセレクタ文字列かDOM要素）。

ハンドル説明
.instance生のコンポーネントインスタンス
.set(props)props を更新
.on(event, cb)コンポーネントの発火を購読
.destroy()破棄（購読・DOMの片付けはエンジン責任）
const app = mount(App, { target: "#app", props: { name: "world" } });
app.set({ name: "bridgey" });
app.destroy();
コンポーネント外へ値を渡すならコールバックpropが便利です（props: { onChange }）。イベントの初期取りこぼしが無く、Svelte/Vue双方で同じに書けます。

部品の書き方

Svelte（.svelte、rollup-plugin-svelte でビルド）:

<!-- App.svelte -->
<script>
  export let name = "world";
  let count = 0;
  $: doubled = count * 2;
</script>
<button on:click={() => count++}>{count}</button>
{#if count >= 10}<p>over</p>{/if}

Vue（SFC、またはビルド不要のフルビルド＋template文字列）:

// App.js
export default {
  props: { name: { type: String, default: "world" } },
  data: () => ({ count: 0 }),
  computed: { doubled() { return this.count * 2; } },
  template: `<button @click="count++">{{ count }}</button>`,
};

useEngine / engine

useEngine(adapter) でエンジンを選択（起動時に一度）。未選択で state/mount を使うと分かりやすい例外を投げます。engine() は内部利用。

import { useEngine } from "bridgey";
import { vueEngine } from "bridgey/engines/vue.js"; // or svelte
useEngine(vueEngine);

$$.ajax / $$.get / $$.post

fetch の薄いjQuery風ラッパ。data がオブジェクトならJSON送信、応答は content-type がJSONなら自動パース、失敗時は例外。

呼び方説明
$$.get(url, options?)GET
$$.post(url, data?, options?)POST（dataオブジェクトはJSON化）
$$.ajax(url, { method, data, headers, type })汎用。type:"text"で常にテキスト

Promise を返すので await / .then でも、jQuery流の .done() / .fail() / .always() でも、コールバック形でも書けます。

// await / then
const users = await $$.get("/api/users");
// jQuery流チェーン
$$.post("/api/todos", { text: "買い物" })
  .done((res) => console.log(res))
  .fail((err) => console.error(err))
  .always(() => done());
// コールバック形($.get(url, fn) の手癖)
$$.get("/api/users", (users) => render(users));

フォーム送信 (serialize)

.serialize() / .serializeArray() で送信対象フィールドをまとめられます（disabled・未チェックは除外、複数selectは複数展開）。

メソッド説明
.serialize()クエリ文字列 "name=太郎&agree=on"
.serializeArray()[{name, value}, …]
$$("#form").on("submit", function (e) {
  e.preventDefault();
  $$.post("/api/contact", $$(this).serialize())
    .done(() => $$("#msg").text("送信しました"));
});

静的ユーティリティ

関数説明
$$.extend(target, …src) / $$.extend(true, …)オブジェクトのマージ（trueで深いマージ）
$$.each(coll, (i, v) => …)配列/オブジェクトの反復（falseで中断）
$$.map(coll, (v, i) => …)変換して配列に（nullは除外）
$$.trim(str)前後の空白を除去

$$.fn（自作プラグイン）

jQueryプラグインの書き味で、自作の拡張を $$ に生やせます。

$$.fn.highlight = function () {
  return this.css("background", "yellow"); // this は $$ オブジェクト
};
$$(".note").highlight().addClass("seen");
既存のjQueryプラグイン(slick / select2 等)はそのままでは動きません。本物のjQuery($.fn.jquery・独自イベント/データ機構・$.Deferred 等)に依存するためです。$$.fn は自作用の入口であって互換層ではありません。既存プラグインは領域を分けて本物のjQueryを併用してください。

npm と <script>（配布）

① npm — import "bridgey"。部品(.svelte/.vue)はビルドします。

② <script> / CDN — ビルド不要で $$ / state / computed / mount / useEngine が window に載ります。2種あります。

ファイル同梱エンジンできること
dist/bridgey.vue.jsVueフルビルドビルド不要でフル機能。template文字列のVue部品を mount まで（Vue公式CDNと同発想）
dist/bridgey.jsSvelte(軽量)$$ / state / computed 中心。mount はコンパイル済みSvelte部品向け
<!-- ビルド不要でモダン部品まで -->
<script src="https://cdn.jsdelivr.net/npm/bridgey/dist/bridgey.vue.js"></script>
<script>
  const App = { data: () => ({ n: 0 }),
    template: `<button @click="n++">{{ n }}</button>` };
  mount(App, { target: "#app" });
</script>

名前の衝突を避ける

Prototype.js や独自定義で $$ を既に使っている場合の対策です。安定した名前空間 window.bridgey は常に残るので、衝突が怖ければ常にこれ経由で使えます（bridgey.$$ / bridgey.state …）。

const b$ = bridgey.noConflict();     // 既存の $$ を復元し、bridgeyの $$ を受け取る
const B  = bridgey.noConflict(true); // 触った全グローバルを復元し api を返す
bridgey.alias("jq");               // window.jq = bridgey の $$（既存名は壊さない）
npm版(import)なら衝突は起きません。名前も自由: import { $$ as $ } from "bridgey"。

CLI: brg init

brg init [dir] [--framework svelte|vue] [--bridgey-dep <spec>]

フレームワークを選ぶと、配線済みのスタータ一式（App.(svelte|js) / main.js / build.mjs / index.html / package.json）を生成します。

エンジンはプロジェクト作成時に一度だけ選びます（後から切り替える機能は持ちません）。別エンジンは別プロジェクトで。利用者コード($$ / state / mount)は共通なので知識はそのまま移せます。

レシピ集

タブ切り替え

$$("#tabs").on("click", ".tab", function () {
  $$("#tabs .tab").removeClass("active");
  $$(this).addClass("active");
});

フォームの送信可否を宣言的に

const valid = computed(() => ok(form.value));
valid.subscribe((v) => $$("#submit").prop("disabled", !v));

APIから取得して一覧描画（命令的）

const users = await $$.get("/api/users");
$$("#list").empty();
users.forEach((u) => $$("#list").append(`<li>${u.name}</li>`));

領域分離で jQueryプラグインと共存

<!-- カルーセルは本物のjQueryの領域 -->
$(".carousel").slick();
// フォームは bridgey の領域（別ノード）
mount(Form, { target: "#form" });

制限・非対応

  • ビルドが基本（例外あり）：Svelte部品はコンパイル前提でビルドが要ります。ただし dist/bridgey.vue.js（Vueフルビルド同梱）なら <script> 一本・ビルド不要で mount まで動きます。
  • jQuery共存はサポートしない：機能として保証・お膳立てはしません。ただし領域を分ければ（例：カルーセル=jQuery / フォーム=Svelte）自己責任で共存可能。禁止は「同一ノードの相乗り」だけです。
  • 既存のjQueryプラグインは動きません：slick / select2 等は本物のjQuery依存です。$$.fn は自作プラグイン用の入口で、既存プラグインの互換層ではありません。
  • 薄いヘルパー：ルーター/スコープCSS等は持ちません（下のFWの領分）。
  • 対象は HTML-first なFW（Svelte / Vue）：React 等の宣言的JSX系は思想が異なるため対象外（命令的DOM操作を避ける設計・単体のリアクティブ原子が無い・JSX前提、と bridgey の狙いが噛み合わないため）。

FAQ

jQueryのコードはそのまま動く？

DOM操作・イベント・走査の大半は $→$$ で動きます。ただしプラグインは別（上記）。$.ajax は $$.ajax（fetchベース）に置き換えます。

state を使わず $$ だけでも使える？

はい。命令的な $$ だけでも使えます。state/computed を使う時だけ useEngine が必要です。

Svelte と Vue、どっちを選ぶ？

どちらでも利用者コードは同じです。テンプレートがHTMLに近いのはSvelte、CDNフルビルドでビルド不要にできるのはVue。迷ったらSvelte（既定）。

後からエンジンを変えられる？

プロジェクト単位で選ぶ設計です（途中変更コマンドは持ちません）。ただし利用者コードは共通なので、新プロジェクトへ移すのは容易です。
